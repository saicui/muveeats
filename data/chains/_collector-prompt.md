```
MuveEats プロジェクトのチェーン店メニュー栄養データを収集する
別働隊として動いてください。

## やること

1. data/chains/_targets.txt にあるターゲット一覧を読む
2. 各チェーンについて、公式の栄養情報ページを WebFetch で取得
   - URL が未記入のものは公式サイトを Web 検索で特定
   - PDF が一次情報の場合はそれを優先
   - 公式から取れないアイテムは記録対象から除外（推測値を入れない）
3. data/chains/README.md のスキーマに従って data/chains/{slug}.json
   を作成・保存
4. 完了したターゲットは _targets.txt の行頭に # を付けてコメントアウト

## ルール

- **公式値のみ**を採用。サードパーティの推測サイトは使わない
- カロリー / P / F / C / 塩分 が揃わない場合は null（空欄）にする
- 全メニューが取れなくても可。代表的な 20-30 品目に絞って良い
- aliases は表記揺れ用（例: "ビッグマック", "BigMac"）
- fetched_at は ISO 日付 (YYYY-MM-DD)
- ファイルが既に存在する場合は上書き前にユーザーに確認

## 優先順位

_targets.txt の上から順に処理。3-5 チェーン進めるごとに状況を
ユーザーに報告して、続けるか確認してください。

## 注意

- このセッションは並行して MuveEats 本体の実装も動いています
- 触っていいのは data/chains/ 配下と _targets.txt のみ
- 他のディレクトリ（app/, lib/ など）には手を出さない

まず最初に _targets.txt と data/chains/README.md を読んで、
スキーマと方針を確認してから着手してください。
```

## セットアップ後の流れ

1. 別チャットで上記プロンプトを実行
2. 進捗報告を受け取りながら品質チェック
3. 全件揃ったらこちらのセッションに戻って、本実装フェーズで
   chain search 機能から `data/chains/*.json` を読み込む実装を入れる

## 並行作業の安全性

- 触るファイル: `data/chains/*.json` と `_targets.txt` のみ
- 本実装側は `app/`, `lib/`, `supabase/` などを触るので衝突なし
- 同じブランチ (main) で進めて OK。コミットだけ分ける
