v8 finalized — フィードバック反映

  • カテゴリ chip 撤廃: 検索画面 (2A) では検索一本に。ユーザーが選んだメニューに応じて、ジャンルタグが選択後画面 (2F) で自動付与される設計に変更。
  • サイズ自動抽出: アイテム名の末尾語（"並盛 / 大盛 / 中盛 / 小盛 / ミニ / Tall / M / L" 等）をパーサで抽出し、本体名と分離して小バッジ表示。例: 「牛丼 並盛」 → name: "牛丼", size: "並盛"。
  • Hot/Iced 抽出: スターバックスは 「スターバックス ラテ（ホット）Tall」 から「Hot」「Tall」の2バッジを抽出。
  • ジャンルタグ（自動） (Screen 2F): 公式値メニューには category から「バーガー / 牛丼・和食 / カフェ / 寿司 / 中華・ラーメン / ピザ / ファミレス / コンビニ」のいずれかを付与。さらにチェーン分類から「ファストフード」「外食」も付与。ユーザーは編集不可（• マーク付きの濃いタグ）。
  • 頭文字ロゴ: 各チェーンの先頭1文字を丸ロゴに。日本語名から取得（マ / モ / ケ / す / 吉 / 松 / ス / ド ...）。SVG ロゴが必要になったら後日差し替え。
  • 並び順: 「牛丼」のような比較系検索は kcal 昇順がデフォルト（Screen 2C 末尾に注記）。クロスチェーン検索（ラテ）はチェーン別グループ表示。

サイズ抽出ロジック（実装メモ）

  • 正規表現で末尾の「サイズ語」を切り出す。優先順位高い順: (ミニ|小盛|並盛|中盛|大盛|特盛|Tall|Grande|Venti|Short|M|L|S)
  • 飲料の温冷指定: (ホット|アイス|Hot|Iced) → 括弧書きの場合も拾う 「ラテ（ホット）Tall」
  • 抽出後の "name" には正規化した本体名のみを格納。サイズはアイテムの size 属性に保存（ DB 検索の表記揺れを吸収）。
  • 抽出できない場合はバッジ表示なし。

ジャンル自動付与のマッピング

  • 各 data/chains/{slug}.json ルートに chain_genre フィールドを追加し、UI 側で参照（例: "chain_genre": ["バーガー", "ファストフード"]）。
  • アイテム単位の category はそのチェーン内分類なので、UI 側マッピング表でジャンル名に変換 (例: KFC の "chicken" → 「バーガー」グループには入らず「ファストフード」のみ)。
  • マッピング表 (data/chain-genres.json として別ファイル化案): slug → ジャンル配列。

本実装で必要なバックエンド処理

  • ビルド時に data/chains/*.json を統合 → data/chains-index.json を出力（全件 + size 抽出済 + ジャンル付与済）。
  • 検索: 料理名 + aliases の部分一致 (ilike)。チェーン名でも一致させる。
  • 「よく使う」: ユーザーの最近 30 件の食事記録から、同一メニュー（chain_id + item_id）を重複排除して上位 3-5 件を表示。
