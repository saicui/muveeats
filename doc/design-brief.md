# MuveEats — Design Brief

> Claude Design へのハンドオフ用ブリーフ。実装済みモックは `mock-v4.html`。
> このブリーフ / `design-system.md` / `screens.md` をセットで参照してください。

---

## 1. プロダクト概要

**MuveEats**（"Move" + "Eats" の造語）は、食事・運動・体組成を**1つに集約する個人ヘルスハブ**。

- **核心思想**: 「動く」と「食べる」は対で意味を持つ。摂取と消費を同じ画面で見られることが価値の中心。
- **ターゲット**: 自分の身体を継続的に管理したい個人ユーザー（筋トレ / 減量 / 健康維持）。専門家ではない一般人。
- **ステージ**: モバイル Web (PWA 化を視野)。スマホ縦持ち前提。
- **AI の役割**: 食事写真→栄養素推定、タグ提案、目標値の相談。「主役」ではなく「裏方」。

---

## 2. デザインの目標 / 非目標

### 目標 (DO)
- **静謐 / 無機質 / プロフェッショナル**: Linear, Notion, Apple Health, Things 3 のような節度のあるトーン。
- **数値が主役**: 摂取 kcal、ボリューム、体重などのデータが読み取りやすいこと。タイポグラフィで「読ませる」。
- **モノクロ + 用途別アクセント**: 黒/白/グレーが土台。役割色は3つだけ（食事 / 運動 / 体組成）。
- **薄い線 / 余白で構造化**: 重い境界線や塗りつぶしは避け、1px の細線と余白でセクションを分ける。
- **片手で使える**: 主要アクションは画面下半分に配置。ボトムナビ前提。

### 非目標 (DON'T)
- ❌ **絵文字を一切使わない**。代替は SVG アイコン（1.5px ストロークの outline 系）。
- ❌ ゲーミフィケーション色 (派手なバッジ、紙吹雪、進捗バーの過剰演出) は避ける。ストリーク等は「数字」だけで十分。
- ❌ グラデーション、シャドウの多用、過度な角丸（≧20px は避ける）。
- ❌ パステル / かわいい寄りの装飾。ただし「Pastel テーマ」はオプションとして用意（後述）。
- ❌ 専門用語の押し付け（"MET" "PFC" などは表示するが、補助説明を添える）。

---

## 3. キーレファレンス

似たトーンの製品（モック作成時の参照）:

- **Linear** — タイポグラフィ、余白、無機質さ
- **Apple Health** — データの並べ方、グラフのミニマリズム、色の使い方
- **Things 3** — 入力フォーム、余白の取り方、グレースケール基調
- **Whoop / Strava (data 画面)** — 数値の見せ方、収支の可視化
- **Notion** — 設定画面の構成、フィールド群の使い方

避けたい例: MyFitnessPal の情報過密、Fitbit の派手な色使い、一般的なヘルスケアアプリのアイコン主体ナビ。

---

## 4. テーマ

3つのテーマを用意。設定画面で切替可能（ヘッダーには切替を置かない）。

- **Light** — 標準。ウォームグレー (`stone` 系)。
- **Dark** — グレー系（純黒ではない）。`zinc-900` 付近をベース。
- **Pastel** — オプション。ピンク～モーブ系の柔らかいトーン。色相を維持しつつ彩度を上げすぎないこと。

すべてのテーマで「無機質さ」を保つこと。Pastel でも、フォントウェイトや余白の規律は崩さない。

---

## 5. 情報設計の原則

- **1画面 1目的**: ダッシュボードでは全体把握、記録画面では入力に集中。混ぜない。
- **進捗・状況は必ず数値**: 「達成率 82%」よりも「目標 120 / 現在 98」を優先。
- **AI 出力は判別可能に**: 「自信度 82%」「破線のチップ = AI 提案」など、AIが介在した情報は人間の確定情報と視覚的に区別。
- **時間は相対 + 絶対の併記**: 「2日前 / 5月23日 07:00」のように。

---

## 6. 想定ユーザーフロー（主要3つ）

1. **食事をログする**: ホーム → 食事を記録 → チェーン検索 or 写真 or 手動 → タグ確認 → 保存
2. **トレーニングをする**: ホーム → 筋トレ → 種目選択 → セット入力（前回参照可） → 完了
3. **進捗を振り返る**: ホーム → 体組成 or 履歴 → 期間切替 → グラフで傾向確認

すべて 5タップ以内で完結することを目安に。

---

## 7. 含めるべきもの / 含めないもの

### 今フェーズに含めるもの (MVP)
- 統合ダッシュボード
- 食事記録（チェーン検索 / 写真 / 手動）
- タグ付け（事前定義タクソノミーから選択）
- 筋トレ記録（セッション + 種目 + セット）
- 有酸素記録
- 体組成記録（手動入力）
- 統合履歴（タイプフィルタ可）
- 設定（目標値 / プロフィール / AI 相談 / テーマ / データ管理）

### 次フェーズ
- バイタル（血圧・心拍）
- メモ / 気分トラッキング
- Apple Health / Google Fit 連携
- 週次 AI レビュー
- PWA 化と通知
- 共有 / エクスポート（CSV はあり）

---

## 8. Claude Design への依頼事項

このブリーフを土台に、以下を期待しています:

1. **mock-v4.html の各画面をブラッシュアップ** — トーンを保ったまま、細部（余白、タイポグラフィ、コンポーネントの呼吸感）を磨く
2. **未モック画面の追加** — `screens.md` に記載した未着手画面（設定画面、ログイン、種目マスタ等）
3. **インタラクションのモック** — ホバー / フォーカス / 遷移の状態を補足
4. **デザインシステムの拡張** — `design-system.md` のトークンを必要に応じて追加（命名は既存に揃える）
5. **必ず守ること**: 絵文字非使用、3アクセント色構造、Light/Dark/Pastel の3テーマ対応

異論・改善案があれば積極的に提案してください。「このアプリは健康管理ハブで、長期的に毎日使うもの」という前提を忘れず、派手さよりも持続的な可読性を優先してください。
