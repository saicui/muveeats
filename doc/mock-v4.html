v4 で変えたこと

  • 絵文字の全撤廃: インライン SVG アイコン（1.5px ストローク）に置き換え。Linear や Apple Health のような無機質な印象に。
  • タイポグラフィ: ヘッダーは大文字 + tracking で「ロゴ的」に、数値は tabular-nums で揃える。Inter 等のグリッド前提フォント。
  • 色: 主役は黒/白/グレーで、用途別アクセントを 3色のみ（食事=深緑、運動=深紫、体組成=テラコッタ）。「ドット」や「3px の細マーカー」で識別、塗りすぎない。
  • RPE 削除: セット入力は重量 × rep の2列のみ。PR は黒バッジで控えめに表示。
  • 有酸素を独立画面化: 種目 / 時間 / 距離 / 心拍 / 強度 を入力 → MET 法で消費 kcal を自動算出。
  • 体組成画面: 体重・体脂肪率を sparkline 風グラフで。筋肉量・内臓脂肪・基礎代謝・BMI も同画面に。
  • 統合履歴: 食事 / 筋トレ / 有酸素 / 体組成 が同じタイムラインに並ぶ。マーカー色で識別。フィルタチップで絞り込み。

健康ハブとしての全体像

  • 記録できるもの: 食事 · 筋トレ · 有酸素 · 体組成 · バイタル (心拍/血圧) · メモ/気分
  • 派生で出せる指標: エネルギー収支 · ストリーク · PR · 平均ペース · 体重トレンド · 1日推奨 kcal
  • AI が出るところ: 写真→栄養素推定 · 自動タグ付け · 目標値の相談 · 週次レビュー（次フェーズ）

チェーン店データ収集の流れ（提案）

  • 実装が一段落したら、別セッションで Claude Code に対し WebFetch + JSON 書き出しのタスクを投げる
  • data/chains/_targets.txt に対象チェーンを列挙 → Claude Code が並列で data/chains/*.json を埋める
  • schema validator スクリプトで値の妥当性をチェック（過剰なカロリーや NULL を検出）
  • 1〜2回の対話で 20〜30 チェーン分のデータが揃う見込み

本実装前の最終確認

  • このトーン（無機質・モノクロ+3色アクセント）で OK？ もう少し温かい印象がいい？
  • 体組成入力は手動で OK（体組成計の数値を打ち込む）か、将来的に Apple Health / Google Fit 連携も視野に入れたい？
  • バイタル（血圧・心拍）は今フェーズで実装する？ それともフェーズ2？
  • 「メモ/気分」も入れる？（食事の満足度・気分トラッキング）
  • これで OK なら、本実装のフェーズ計画を立てて着手します。
