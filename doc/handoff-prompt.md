# Claude Design 開始プロンプト

`doc/` 配下を全て読み込ませた後、目的に応じて以下のいずれかを送ってください。

---

## A. まず全体把握 + 提案をもらいたい（推奨）

```
あなたは MuveEats のデザインパートナーです。

事前に読み込んだ doc/README.md, design-brief.md, design-system.md,
screens.md, mock-v4.html を MuveEats プロジェクトの一次資料として
扱ってください。

まずは以下の3つを返してください:

1. 現状の mock-v4.html について、デザインブリーフのトーン
   （無機質 / モノクロ + 3アクセント / 絵文字なし）に照らして
   気になる点を 5つまで指摘してください。重要度順で。

2. screens.md で ☐ になっている未モック画面のうち、
   どの順番で作るべきか優先度の根拠とともに提案してください。

3. デザインブリーフや design-system.md に追加 / 修正したほうが
   良い項目があれば挙げてください。

実装に入る前に、上記3点で方針合意を取りたいです。
```

---

## B. 特定の未モック画面を作ってほしいとき

```
MuveEats のデザインを進めます。

doc/ 配下の design-brief.md / design-system.md / screens.md /
mock-v4.html を一次資料として扱ってください。

screens.md の以下の画面を作成してください:

- Settings (Index)
- AI Consultation ボックス
- Meal Detail Modal

要件:
- mock-v4.html と同じ単一 HTML ファイルとして出力
- design-system.md のトークン (CSS 変数) を流用、必要なら追加
- Light / Dark / Pastel の3テーマすべてに対応
- アイコンは mock-v4.html の SVG symbol を使い、足りないものは
  同じスタイル (1.5px stroke / outline) で追加
- 各画面に「空状態」「ローディング」のバリエーションも添える

完成後に、追加 / 変更したトークンと、判断に迷った点を
箇条書きで報告してください。
```

---

## C. 既存モックをブラッシュアップしてほしいとき

```
MuveEats の mock-v4.html をブラッシュアップしてください。

design-brief.md のトーン (無機質 / モノクロ + 3アクセント / 絵文字なし /
Linear・Apple Health・Things 3 のテイスト) を保ったまま、以下の観点で
磨いてください:

1. 余白とリズム — 各セクションの呼吸感、視線誘導
2. タイポグラフィの階層 — Section title / Stat / Body の見え方
3. コンポーネントの一貫性 — Card, List Row, Field Group の統一感
4. インタラクションの状態 — Hover, Focus, Active, Empty, Loading

成果物:
- 改訂版 HTML (単一ファイル)
- 変更点のサマリー (どこを / なぜ変えたか)
- 採用しなかった代替案があればその理由

design-system.md にトークン追加が必要な場合は、命名規則を揃えて
提案してください。
```

## 補足の伝え方（必要に応じて末尾に追加）

- **時間の制約**: 「30分以内で全体提案だけ欲しい」「腰を据えて全画面やってほしい」など
- **重視するテーマ**: 「特に Dark テーマでの可読性を見てほしい」
- **避けたい要素**: ブリーフに無い NG があれば明示
- **既存実装との整合**: 「実装フェーズで再利用するので、CSS 変数とクラス命名は変えないで」

## 1ターン目で確認しておきたいこと（チェックリスト）

Claude Design が応答してきたら、次の点を確かめると後工程がスムーズです:

- [ ] design-brief.md の「非目標」を理解しているか（絵文字 / ゲーミフィケーション排除）
- [ ] 3アクセント色（eat / move / body）の使い分けを正しく踏襲しているか
- [ ] 提案がトークン化されているか（ベタ書きの色や余白がないか）
- [ ] Light / Dark / Pastel の3テーマすべてに対応しているか
- [ ] モバイル前提 (max-width 440px) を守っているか
