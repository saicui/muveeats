## ☑ Screen 1: Dashboard (Home)

**目的**: 今日 1日の摂取・消費・体組成を一望。各機能への入口。

**構成要素**:
- ヘッダー: BRAND + メニュー
- 大見出し: 今日の日付
- INTAKE / BURN の双子カード（数値強調 + メタ）
- ENERGY BALANCE バー（双方向）+ 純摂取
- BODY サマリーカード（最新の体重と体脂肪率）
- 直近 7日のドット（食事/運動/体組成の有無）
- Quick actions グリッド（6マス：食事/筋トレ/有酸素/体組成/バイタル/メモ）
- 記録ストリーク

**インタラクション**:
- 双子カードをタップ → 該当ドメインの今日の詳細
- Quick action タップ → 対応する記録画面

## ☑ Screen 2: Record Meal

**目的**: 食事を1つ記録する。チェーン店検索を最優先導線に。

**構成要素**:
1. **チェーン店から検索** (最上部、search input + 結果リスト)
   - 結果には `official` バッジ or `推定値` 表示
   - メニュー名 / 店名 / kcal / PFC
2. 写真から | 手動入力 の2ボタン（チェーン検索を選ばない場合の代替）
3. 選択した内容を編集できるフィールド群
4. PFC 4セルの数値表示
5. タグ選択（事前定義のカテゴリ別）
   - AI 提案は破線で示す
6. 「記録する」CTA

**バリデーション**:
- 料理名は必須
- kcal は数値、空欄なら 0 として扱う

---

## ☑ Screen 3: Strength Session (active)

**目的**: 筋トレセッションを記録しながら進める。

**構成要素**:
- ヘッダー: セッション名 + 経過タイマー
- インターバルタイマー（次セットまでのカウントダウン）
- 種目カード:
  - 種目名 + 前回参照
  - セット表 (SET / WEIGHT / REP / 完了チェック)
  - 「セットを追加」ボタン
- 「種目を追加」破線ボタン
- 中断 / セッション完了 CTA

**ルール**:
- RPE は表示しない（削除済）
- PR は黒の小バッジで控えめに
- 前回値を opacity で薄く表示（再現したいなら）

---

## ☑ Screen 4: Cardio Record

**目的**: ラン / ウォーク / バイク等の有酸素記録。

**構成要素**:
- 種目セレクター（4ボタン：ラン/ウォーク/バイク/その他）
- フィールド群:
  - 開始時刻
  - 時間 (分)
  - 距離 (km)
  - 平均心拍 (任意)
  - 強度 (軽い/中/強い)
- 自動計算ブロック:
  - ペース ('/km)
  - 消費 kcal (MET 法)
- MET 法の補足説明
- メモ欄
- 「記録する」CTA

---

## ☑ Screen 5: Body Composition

**目的**: 体重・体脂肪率の推移と派生指標。

**構成要素**:
- 体重 sparkline カード (期間切替: 1W/1M/3M/6M/1Y/All)
- 体脂肪率 sparkline カード
- 派生指標 2×2 グリッド: 筋肉量 / 内臓脂肪 / 基礎代謝 / BMI
- 記録履歴リスト

**入力**: ヘッダー右の `+` から記録モーダル（Screen 10）を開く。すべて手動入力（体組成計の数値を読んでそのまま打ち込むフロー）。

---

## ☑ Screen 6: History (Unified Timeline)

**目的**: 全ドメインを統合した時系列ログ。

**構成要素**:
- フィルタチップ (すべて/食事/筋トレ/有酸素/体組成)
- 日付ブロック:
  - 日付見出し + その日の摂取/消費合計
  - リスト行（マーカー色で識別）
- 「もっと見る」ボタン（追加ロード方式）

**行の表示要素**:
- 食事: 料理名 / 時刻 / 自信度 or 手動 / kcal / PFC / タグ
- 筋トレ: セッション名 / 時刻 / 時間 / 消費 kcal / PR有無
- 有酸素: 種目 / 時刻 / 距離 + 時間 + 心拍 / 消費 kcal / ペース
- 体組成: 体重 + 体脂肪 / 時刻 / すべて手動記録

## 🆕 Screen 7: Settings (v5 で実装)

**目的**: 目標設定 + プロフィール + AI 相談 + 表示設定 + データ管理。

**構成案**:

### Settings/Index
- 上部に AI 相談ボックス（プロフィールから目標値を提案）
- セクション:
  - 1日の目標 (kcal / P / F / C)
  - プロフィール (性別 / 年齢 / 身長 / 体重 / 活動レベル / 目標)
  - 表示 (テーマ / ストリーク表示)
  - データ (CSV エクスポート / アカウント削除)
  - サインアウト

### AI Consultation ボックス
- メッセージスタイル: User吹き出し（右） / AI吹き出し（左）
- AI からの提案には「この値で設定する」ボタンを添える（ワンタップで目標欄に反映）
- 過去のやりとりは保存しない（揮発的）— または最新の1ターンだけ表示

## 🆕 Screen 9: Meal Detail Modal (v5 で実装 / v6 で編集可に)

**目的**: 履歴で食事をタップしたときの詳細 + 微修正。

**構成要素**:
- ボトムシート型（下からスライドイン）
- ハンドル (4px 高さの灰バー)
- タイトル: 料理名 / 永続メタ: 日時 · 記録ソース · 自信度（AI 色）
- **編集可能フィールド** (v6):
  - 料理名 / 食べた時刻 / kcal / P / F / C
- タグ選択（予め選択はそのまま、タップで ON/OFF）
- AI メモ（読み取り専用 / 写真記録の場合のみ）
- アクション: 削除 / キャンセル / 保存

**挙動**:
- 保存した値は「手動修正された記録」として保持される（AI 認識値との差分はログにのみ保持）。
- タグはユーザーが手動選択 / 解除したとされ、AI 提案フラグ（破線）は以降付かない。

**目的**: 履歴で食事をタップしたときの詳細。

**構成要素**:
- ボトムシート型（下からスライドイン）
- ハンドル (4px 高さの灰バー)
- 料理名 / 日時 / 記録ソース
- PFC 4セル
- タグ群
- AI メモ（写真記録の場合）
- アクション: 削除 / 閉じる
  - 編集はなし（削除して再記録するフロー）

## 🆕 Screen 10: Body Composition Entry Modal (v5 で実装)

**目的**: 体重・体脂肪などの手動入力。

**構成**:
- ボトムシート
- 日時 / 体重 / 体脂肪率 / 筋肉量 / 内臓脂肪 / 基礎代謝を縦に

## 共通の作成方針（Claude Design へ）

- すべての画面で**ヘッダー + コンテンツ + ボトムナビ**の構造を保つ
- max-width: 440px のフレーム内に収める
- 各画面の上部に**サマリー / 主要数値**、中央に**コンテンツ**、下部に**CTA**を置く
- 「絵文字を入れたくなる」場面は SVG アイコンに置き換える
- ローディング / 空状態 / エラー状態のバリエーションも合わせて作成すること
